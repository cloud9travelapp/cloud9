# Project handoff ‚Äî Cloud9

AI travel-planning web app. Next.js 16 (App Router, Turbopack) ¬∑ React 19 ¬∑ TypeScript ¬∑ Tailwind v4 ¬∑ NextAuth v5 (Google ‚Üí Supabase) ¬∑ Anthropic SDK (claude-sonnet-5 concierge, claude-haiku-4-5 namer). Repo `cloud9travelapp/cloud9`, deploys to Vercel from `main`. Mock providers need no env vars.

## Architecture (key patterns)
- **Chat**: streaming route `app/api/chat/route.ts`. The system prompt IS the concierge's behavior spec. Tool round-trip loop supports multiple tools; auth-gated; saves to Supabase. Model history window = the LATEST 40 messages (desc + reverse ‚Äî taking the oldest was the "context bleed" bug; same for the 200-message display window in `app/chat/page.tsx`). Pre-tool text is declared USER-VISIBLE in the prompt (tool_choice:auto lets the model emit chain-of-thought before tool_use ‚Äî the documented, prompt-side-handled narration-leak cause; remember_preference calls carry no text at all).
- **Runtime diagnosis = SQL, not logs**: `lib/diag.ts` `logDiag(kind, detail)` ‚Üí Supabase `diag_events` (best-effort, console-mirrored). Instrumented: every tool_call (+outcome), search errors (real cause), stays quota fallback, stream errors, title errors. Vercel's ~1h log retention makes log-hunting a dead end ‚Äî query `diag_events` instead. Extend this pattern for any new "what happened at runtime?" question.
- **Trip titles**: `deriveTripTitle` (haiku) runs each turn via `after()` (never blocks the stream) ‚Äî returns a title covering ALL destinations ("Japan & Korea") or KEEP (defined strictly: nothing new named, nothing changed ‚Äî the old "no destination known yet" phrasing made haiku KEEP first turns). While a trip is "New Trip" the titler sees the last ~5 user messages (self-healing). A user rename (pencil in the sidebar ‚Üí `PATCH /api/trips/[id]`) sets `name_is_custom`, which locks the title against auto-updates. Both paths degrade gracefully if the `name_is_custom` column is missing. Titles use the SHORTEST natural name (never "City, Country"; origin city is never a destination) and pass through the deterministic `sanitizeTripTitle` guard (`lib/trip-title.ts`, unit-tested) before storage.
- **Prompt caching**: the concierge system prompt = `systemStatic` (persona + prefs + date + ALL behavior rules; `cache_control: ephemeral`; byte-identical across a user's turns) + `systemDynamic` tail (per-turn language directive, first/returning line). NEVER interpolate per-turn text into the static block ‚Äî it kills the cache. Each turn logs `chat timing: pre-model/first-token/total` to the runtime logs.
- **Reply language is computed, never model-inferred**: `lib/language.ts` decides each turn's language (word-dominance of the latest user message; place/brand names don't flip it; ambiguous ‚Üí conversation's established language; new+ambiguous ‚Üí Hebrew, the onboarding-preference seam) and the route injects it as a hard per-turn directive covering pre-tool notes, tool summaries, and block strings. Don't add prompt language rules that re-infer from "the user's message" ‚Äî that was the root cause of the mixed-language bugs.
- **Provider-agnostic search** (real provider = one-file swap): `lib/flights/*` (`search_flights`) and `lib/stays/*` (`search_stays`). Each = types + a `searchX` switch on `X_PROVIDER` env (default `"mock"`, exports `IS_MOCK_*`) + a seeded mock. Don't change existing provider signatures.
- **Known demo limitation ‚Äî TEST-environment inventory tops out at ~4‚òÖ** (confirmed 2026-07-21: the cached Paris fetch's star histogram had no 5‚òÖ; our tercile/star filtering was clean). The luxury band can't show what Hotelbeds test inventory doesn't carry; the prompt's inventory-honesty rule attributes the gap to "my current inventory", never to the city. Re-check when production keys arrive.
- **Hotelbeds provider (LIVE since 2026-07-20)**: `lib/stays/hotelbeds.ts` ‚Äî APItude geolocation availability search (the concierge passes lat/lng in the tool call, like IATA codes); active via `STAY_PROVIDER=hotelbeds` + `HOTELBEDS_API_KEY/SECRET` in Vercel (base URL defaults to the test host). Evaluation tier = 50 req/day: results cached in Supabase `stay_search_cache` (24h TTL), 45-call/day guard falls back to mock whose `mock-` ids re-label cards as test data. v1 is availability-only for amenities/photos; distance-from-center is COMPUTED (haversine vs the searched point, on cards + in offers JSON for the model). Result pipeline (all unit-tested): PRICE-first budget terciles (‚â•4‚òÖ guard on luxury ‚Äî real categories don't track price, "‚Ç¨42 4‚òÖ") ‚Üí median-relative DISTANCE tier (near half of the set, nearest-first backfill to 5; `distanceFilter:"any"` tool param skips; user-named area moves the searched point itself) ‚Üí 8-card cap ‚Üí worth-it DEAL detection (far offer ‚â•30% under shown same-star median, split into the tool JSON as `deal`, offered via one sanctioned teaser sentence ‚Äî the only question allowed in a cards message).
- **Preferences (ask-don't-assume)**: the concierge records ONLY explicitly-stated/confirmed preferences via the `remember_preference` tool ‚Äî scope `stable` ‚Üí `users.preferences` (crosses trips, in the static prompt's profile line), scope `trip` ‚Üí `trips.preferences` jsonb (dynamic tail; dies with the trip). Known preferences open as questions ("◊í◊ù ◊î◊§◊¢◊ù?"), never silent filters. The old keyword auto-learner (`lib/preferences.ts`) is deleted ‚Äî do not reintroduce inference-based preference learning.
- **Delimiter blocks**: the model appends ONE `<<OPTIONS>> | <<FLIGHTS>> | <<STAYS>> ‚Ä¶ <<END>>` block per message. Frontend `components/chat/chat-client.tsx` parses via `displayText` (strips from the first `<<`, so raw blocks never leak) + tolerant `blockRaw`. Card views live in `components/chat/message-parts.tsx` (shared by the real chat AND the landing demo, so they can't drift). Shared `CardSelect` action posts a structured choice as the user's message.
- **Design system**: time-of-day phase tokens (`--c-*`) under `[data-phase]` in `app/globals.css`; 5 phases (sunrise/morning/midday/sunset/night) set from local time (flash-free via a `next/script` inline init). Suez One (display) + Heebo (body). Chat is RTL-first with per-message `dir="auto"` + `unicode-bidi: plaintext`; times/codes/prices stay LTR. Every animation is gated under `prefers-reduced-motion`.
- **Brand**: `components/brand/` (cloud mark + `Lockup`); favicon `app/icon.svg` + `app/apple-icon.tsx`; social image `app/opengraph-image.tsx` (fixed sunset palette).
